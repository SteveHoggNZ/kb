## New Notes Added

### [[concepts/design-principles/Commands That Emit Events|Commands That Emit Events]]
Unifying philosophy for reactive API design: every API endpoint is a **command** expressing business intent that **emits events** as a side effect. Stop thinking "endpoints that mutate state" — start thinking "actions that announce what happened." Introduces **Action URLs** (hybrid REST with intent-based endpoints like `POST /orders/{id}/ship`), **three levels of decoupling** (temporal, spatial, protocol), the **Reactive Contract** (The Trifecta: document actions, events, and subscriptions together), and **event versioning strategy** (stable types, versioned payloads, additive-only changes). Synthesises existing CQRS, Event-Driven Side Effects, Outbox Pattern, and Async-First Design notes into a coherent architecture.

### [[concepts/ai-development/Intent Engineering|Intent Engineering]]
The missing discipline above Context Engineering. Three eras: Prompt Engineering → Context Engineering → Intent Engineering. Context tells the AI what to know; Intent tells it what to want. The **Intent Gap** (disconnect between org goals and AI metrics) requires three layers: **Layer 1** unified context infrastructure (cures Shadow AI — agents *act on* data, not just access it), **Layer 2** coherent AI worker toolkit (AI Activity ~30% gains vs AI Fluency ~300% gains), **Layer 3** the goal translation layer with the **Cascade of Specificity** (Goal Structures → Delegation Frameworks → Feedback Loops). Delegation Frameworks are the safety net: trade-off hierarchies, decision boundaries, escalation paths — decomposing human principles like "Customer Obsession" into machine-readable rules. Two case studies: **Klarna** and **Microsoft Copilot**. Full architecture Mermaid diagram included.

### [[concepts/design-principles/Vibe Code to Production UI|Vibe Code to Production UI]]
Practical checklist for upgrading AI-generated interfaces into professional SaaS products. Eight predictable AI UI mistakes and their fixes: emojis → icon libraries (Phosphor/Lucide), clashing colours → constrained palettes, KPI boxes → micro-charts with trends, sprawling layouts → condensed with pop-overs and contextual menus, flyouts → centred modals with progressive disclosure, billing/analytics/landing page overhauls. Meta-pattern: AI handles Type 1 (functional); human applies Type 2 (trust, cohesion). Includes quick-reference checklist.

### [[concepts/ai-development/Agentic Primitives|Agentic Primitives]]
Three building blocks that move AI from "pseudo-work" (summarising, briefing — still leaves you reading) to "real work" (tasks genuinely off your desk). **Scheduled Tasks** (computer in the cloud — recurring background work while you sleep), **Dispatch** (mobile orchestration of parallel agent sessions — spawn agents from your phone), **Computer Use** (rescuing trapped data from legacy systems with no APIs). Together they push toward [[Agent Autonomy Levels|Level 5]] autonomy. Requires the **Manager Mindset**: delegate commitments immediately, gather deep context before deciding, trust the background process. Self-hosted vs managed infrastructure analogy: Gmail replaced self-hosted mail, AWS replaced rack servers, managed AI orchestration enables mass adoption.
