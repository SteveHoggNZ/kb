## New Notes Added

### [[concepts/design-principles/Commands That Emit Events|Commands That Emit Events]]
Unifying philosophy for reactive API design: every API endpoint is a **command** expressing business intent that **emits events** as a side effect. Stop thinking "endpoints that mutate state" — start thinking "actions that announce what happened." Introduces **Action URLs** (hybrid REST with intent-based endpoints like `POST /orders/{id}/ship`), **three levels of decoupling** (temporal, spatial, protocol), the **Reactive Contract** (The Trifecta: document actions, events, and subscriptions together), and **event versioning strategy** (stable types, versioned payloads, additive-only changes). Synthesises existing CQRS, Event-Driven Side Effects, Outbox Pattern, and Async-First Design notes into a coherent architecture.
