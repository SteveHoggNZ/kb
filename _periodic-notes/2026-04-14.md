- [[Structure-Preserving Data]] — First-class principle extracted from Agent Memory Architecture: the structure of a document *is* information — flattening it into text chunks for vector search is lossy compression with no recovery path. Tables, section hierarchies, forms, graphs, and timelines all carry meaning in their shape. Agents act on what they receive with no human to compensate for structural loss, making this Physics not a nice-to-have. Connected to AMP frames (structured end-to-end), [[Contextual Breadcrumbs]] (typed metadata, not narrative), [[Strategic vs Tactical Documentation]] (tactical docs are inherently structured), and [[Law vs Physics in Agent Design]].

- [[Agent Memory Architecture]] — Why RAG breaks for agents: chatbots need related text, agents need structured operating context (intent, policies, provenance, confidence). Three principles: shape data to match business usage, retrieval units must match the work, define the contract before choosing infrastructure. The **Bundle Pattern**: a specific manifest of every data piece per workflow, not vague "relevant context." Three steps: contract → bundle → primitives. Cross-linked with [[Outcome Agent Architecture]] (deepens Layer 1), [[Conway and Intelligence Portability]] (structured bundles as portable memory vs opaque lock-in), [[Intent Engineering]], [[Agentic Context Framework]], and [[Agents vs Long Context]].

- [[The Implementation Layer]] — The enterprise moat is not the model, data, or API — it's the custom harness making a model enterprise-grade. Six components: Workflow Design, Data Access, Authority Frameworks, Evals, Audit Trails, Recovery & Ownership. Four competitive axes squeezing startups: frontier labs down-stack, consultancies up-stack, systems of record exposing APIs, PE as distribution. "Generic SaaS all tasted like chicken" — agentic value is the opposite. Bridges [[The Bottleneck Economy]]'s Integration Gap, scales [[Harnesses Matter More Than Models]] to enterprise strategy, maps the six components to [[Agentic Harness Primitives]].

- [[Scaling Agents in Large Codebases]] — Seven tactics for building an AI layer in massive repos: lean layered CLAUDE.md rules, scoped working directories, start/stop hooks (auto-distillation), path-scoped skills, LSP via custom MCP servers, sub-agents for exploration vs editing, organisational adoption via champion team. Core principle: context discipline scales, context overload doesn't. Cross-linked with [[Decompose-Route-Recompose]], [[Context Distillation Loop - amnesia as a feature|Context Distillation Loop]], [[Agentic Context Framework]], [[Strategic vs Tactical Documentation]], and [[The Implementation Layer]].
