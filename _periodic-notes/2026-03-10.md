I'm not sure about the strategic value of AMP anymore. I think the main thing that customers struggle with is agent sprawl and tracking which agents are deployed where to which platform, but not necessarily for those agents to be able to talk with each other. I think there are probably good ideas in AMP that are useful.

For example, having a shared history for all work that agents do, I see as valuable. What else could there be in AMP that it can provide that other platforms can't? For example, I see ourselves competing with Microsoft Copilot or Salesforce Agent Force or AWS and Bedrock Agent Core and their Quick platform. All of these things probably do a better job of orchestrating agents when those agents are within the platform itself. And so trying to use AMP for orchestrating agents that are already in the same platform is not a strategic approach. For example, one customer has a bunch of information in Salesforce and other data sources like Snowflake and the AWS consultant said to them that they should just use AWS Quick and that Quick integration with Salesforce and with other data platforms is the correct approach.
