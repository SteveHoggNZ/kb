## New Notes Added

### [[concepts/ai-development/Trinity Protocol|Trinity Protocol]]
Multi-agent coordination framework built on three primitives: **Semaphore** (state — `_data` frames), **Bell** (signal — mentions), **Watchdog** (entropy — timers). One Law: "Show and Tell" — patch state AND mention next agent. Physics backstops: Watchdog timers fire if no mention arrives, Sticky Streams auto-coalesce entries within 3s window. Born from a real failure where SSE-enabled agent skipped mention and webhook-only Commander never woke up.

### [[concepts/design-principles/Logical Resource Separation|Logical Resource Separation]]
One env var per concern, not one env var per server. Even when sharing one physical Redis/DB/S3, give each logical function its own connection string. Future scaling becomes config, not code. Extracted from Redis configuration review — applies to databases, buckets, API keys, anything where two features would benefit from different scaling policies.

### [[concepts/design-principles/Agent Ergonomics|Agent Ergonomics]]
Cockpit shape coding applied to software. Aircraft controls use different shapes, colours, and textures so pilots can identify them without looking — three senses converge to make the correct action intuitive. Introduces **Affordance** as the middle layer between Law and Physics. Two dimensions: **Human→Agent** (interaction modes, feedback shapes, error recovery) and **Agent→Tool** (distinct tool names, parameter shapes, response structures). Includes the Blindfold Test, Swap Test, and Pressure Test.

### [[concepts/ai-development/Contextual Breadcrumbs|Contextual Breadcrumbs]]
Solve LLM context amnesia by injecting reminders into tool results — the last thing an agent reads before its next thought. Borrows **stigmergy** from biology: agents leave chemical trails in the environment. Three primitives: `on_turn_reminder` (constant whisper), `on_close_reminder` (post-close instruction), `require_collection_status` (blocking validation). Agents write their own breadcrumbs — dynamic working memory that the platform echoes back at perfect recency.

### [[concepts/design-principles/Dumb Pipe Smart Edge|Dumb Pipe, Smart Edge]]
Architectural principle: keep your API as a dumb pipe (stores and returns data, no opinions). Push Physics to the database (constraints, RLS). Push intelligence to the edge layer (MCP — interprets metadata, injects breadcrumbs, enforces protocols). Push autonomy to the agent (writes own reminders). The Sandwich Rule: Physics sinks to the bottom, intelligence rises to the top, the middle stays dumb. Each layer maps to the enforcement spectrum: Database = Physics, API = neutral, Edge = Ergonomics, Agent = Law.

### [[concepts/ai-development/Seven Dimensions of Hard Work|Seven Dimensions of Hard Work]]
"Hard work" is not monolithic — it splits into seven dimensions, each with a different automation trajectory. **Reasoning** (deep logic) → solved by pure intelligence. **Effort** (volume) → solved by agents. **Coordination** (alignment) → emerging. **EQ** (human dynamics) → human domain. **Judgment/Courage** (nerve to execute) → human domain. **Domain Expertise** (pattern matching) → contested. **Ambiguity** (defining the problem) → human domain. Refines Barbell Economy's Layer 2 into four distinct capabilities; clarifies that taste lives at the intersection of Dims 5-7.

### [[concepts/ai-development/Decompose-Route-Recompose|Decompose-Route-Recompose]]
The meta-skill for the AI era: break complex work into sub-problems by dimension, route each to the right engine, recompose with human judgment. Includes Dex Horthy's brownfield application — **Research-Plan-Implement** — where each phase gets a fresh context window and produces a dense handoff artifact (`research.md` → `plan.md` → code). Key insights: the "Dumb Zone" (>40% context = degraded reasoning), sub-agents as context isolators not personas, review shifts from code to plans, and intentional compaction as operationalised Context Distillation.
