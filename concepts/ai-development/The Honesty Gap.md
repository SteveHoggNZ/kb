# The Honesty Gap

As AI models get more intelligent, their willingness to admit when they don't know something decreases. They have a strong desire to please the user, so they guess rather than admit ignorance.

This is compounded by **Automation Bias**: because the AI sounds confident, users trust it more and check its work less. The result is compounding errors, especially in extraction tasks (contracts, meeting notes, receipts) where the AI infers or hallucinates an answer rather than admitting it can't find one.

---

## Why Confidence Scores Don't Work

The instinct is to ask the model to rate its own confidence — similar to the self-rating trick from [[Thinking Out Loud]] ("Rate that out of 10"). But there's a critical difference between creative/drafting tasks and extraction tasks.

For **creative work**, self-rating is a cheap quality multiplier. The model identifies its own shortcomings and iterates. The "lie" doesn't matter much because you're using the rating to trigger improvement, not to establish ground truth.

For **extraction/verification work**, self-rating gives the agent another opportunity to lie. A model that hallucinated an answer will confidently rate that answer 9/10. The confidence score is generated by the same process that produced the error — asking for a score doesn't create independent verification. You need a different mechanism.

---

## Three Rules to Stop AI Guessing

### Rule 1: Force Blanks, Require Explanations

Don't ask for confidence — force a structural signal. Instruct the model to leave fields **BLANK** when values are ambiguous, missing, or unclear. Require a separate "Reason" column explaining why each blank was left blank, quoting or referencing the specific section of the document.

**Why it works:** Blanks are scannable. The human can quickly spot gaps and read the agent's reasoning to make the final call. The agent can't hide uncertainty behind confident prose.

### Rule 2: Penalise Guessing

By default, a model treats a wrong answer and a blank answer as equally bad. Change the incentive:

> *"A wrong answer is 3x worse than a blank. When in doubt, leave it blank."*

This is [[Conditioning the Distribution]] through incentive design — you're shifting the model's output distribution away from confident guessing toward honest uncertainty. Simple instruction, significant behavioural change.

### Rule 3: Label Sources (Extracted vs Inferred)

Even with strict extraction instructions, models drift toward inference on complex tasks. Create a safety net:

Add a "Source" column. Every value must be labelled:
- **EXTRACTED** — directly stated in the document, exact match
- **INFERRED** — derived from context, calculated, or interpreted

For every INFERRED field, require a one-sentence explanation of the basis.

**Why it works:** The human can trust EXTRACTED data and focus review time only on INFERRED fields. This drastically reduces the cognitive burden of checking the AI's work — the same burden-shifting problem identified in [[Framing the AI Slop Problem]], but solved structurally.

## The Combined Prompt

```
Extract the following fields from this document into a table.

Rules:
Only extract values that are explicitly stated in the document.
When a value is ambiguous, missing, or unclear, leave the FIELD BLANK.
A wrong answer is 3x worse than a blank. When in doubt, leave it blank.

For each field, add a column called "Source." Mark each value as one of:
EXTRACTED: directly stated in the document, exact match
INFERRED: derived from context, calculated, or interpreted

For every INFERRED field, include a one-sentence explanation of what you based it on.
For every BLANK field, add a row to a separate "Flags" table explaining why the value could not be extracted.
```

## Connection to Other Concepts

This is [[Physics Thinking]] applied to prompt design. Rather than relying on the agent's self-assessment (law — it can choose to lie), you create structural mechanisms (physics — blanks, labels, flags) that make uncertainty visible regardless of the model's inclination to please.

The pattern also connects to [[Harnesses Matter More Than Models]]: the linting feedback loop works because the linter creates an independent verification signal. Similarly, the EXTRACTED/INFERRED labelling creates a structural signal that doesn't depend on the model's honesty about its own confidence.
